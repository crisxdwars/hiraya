Controlling Your Loops

Sometimes you don't want a loop to run to completion. You might need to exit early or skip a specific iteration. Java provides two keywords for this level of control: `break` and `continue`.

The `break` Statement: Emergency Exit

The `break` statement is like an emergency exit for your loop. As soon as Java encounters `break`, it immediately terminates the loop and continues execution at the next statement after the loop block.

This is extremely useful when you're searching for something and want to stop as soon as you find it.

String[] treasureChest = {"Gold Coin", "Old Map", "Magic Gem", "Rusty Key"};
String itemToFind = "Magic Gem";

for (String item : treasureChest) {
  System.out.println("Searching... found a " + item);
  if (item.equals(itemToFind)) {
    System.out.println("Found it! Stopping the search.");
    break; // Exit the loop immediately
  }
}
Output:
Searching... found a Gold Coin
Searching... found a Old Map
Searching... found a Magic Gem
Found it! Stopping the search.

The `continue` Statement: Skip This One

The `continue` statement is different. Instead of terminating the loop, it just skips the current iteration and moves on to the next one.

This is useful when you want to process most items in a loop but need to ignore certain ones that meet a specific condition.

// Process all enemies in a level, but skip the friendly ones
String[] enemies = {"Goblin", "Friendly NPC", "Dragon", "Slime"};

for (String enemy : enemies) {
  if (enemy.equals("Friendly NPC")) {
    System.out.println("(Skipping friendly target...)");
    continue; // Skip the rest of this iteration and go to the next enemy
  }
  System.out.println("Attacking " + enemy + "!");
}
Output:
(Skipping friendly target...)
Attacking Goblin!
Attacking Dragon!
Attacking Slime!
