Choosing the Right Container

Think of data types like storage boxes of different sizes and shapes. You need to choose the right box for the type of data you want to store. Java requires you to be specific about this.

These types are split into two main categories: primitive and non-primitive (or reference) types.

Primitive Types: The Basic Boxes

Primitive types are the most basic data types available in Java. They are like simple, pre-built boxes designed to hold one specific kind of value.

  • For Numbers: You have different sizes. Use int for most whole numbers (like a player's score or level). If you need to store a massive number (like the population of the galaxy), you'd use a bigger box called long. For numbers with decimal points (like a price or a percentage), you'd use double.
  • For Single Characters: To store just one letter or symbol, you use the char type.
  • For True/False: To store a simple "yes" or "no" value, you use boolean.

Let's see this in action with a game character example:

// Storing stats for a game character
int playerLevel = 10;
double healthPercentage = 95.5;
char rank = 'A';
boolean isPoisoned = false;

System.out.println("Level: " + playerLevel);
System.out.println("Health: " + healthPercentage + "%");
Output:
Level: 10
Health: 95.5%

Non-Primitive Types: Complex Containers

Non-primitive types, also called reference types, are more complex. Instead of holding the data directly, they hold a reference (like a memory address) to where the actual data is stored. The most common one you'll use is String.

While a char can only hold one letter, a String can hold a whole sequence of them. Because they are more complex, they start with a capital letter (e.g., `String`, not `string`).

String playerName = "Astra";
String guild = "The Night Owls";

System.out.println("Player: " + playerName + " of " + guild);
Output:
Player: Astra of The Night Owls
