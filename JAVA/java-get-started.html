The `main` Method: Your Program's Engine

Think of a Java program as a spaceship. To launch it, you need to start its engine. In Java, the engine is a special method called `main`. It's the very first piece of code that runs when you execute your program.

public static void main(String[] args) {
  // All your mission commands go here...
}

This `main` method is always located inside a `class`, which acts as the blueprint for your spaceship.

The Golden Rule: Matching Filename and Class Name

This is the most important rule for beginners in Java. If your spaceship's blueprint is `public`, its name must exactly match the name of the file it's saved in. This includes capitalization.

If you have a `public` class named `Starship`, the file **must** be saved as `Starship.java`.

// This code must be in a file named "Starship.java"
public class Starship {
  public static void main(String[] args) {
    System.out.println("Engine started. Ready for launch!");
  }
}
Output:
Engine started. Ready for launch!

If you save this as `starship.java` or `MyShip.java`, the Java compiler will refuse to build it.

What if a Class Isn't `public`?

A Java file can contain multiple class blueprints, but only one can be `public`. If a class is not `public`, it's like a secondary system on your spaceshipâ€”it can only be accessed by other parts within the same system (or "package").

If your file contains **no public classes**, you are free to name the file anything you want.

// This file can be saved as "SystemsCheck.java"
class EngineControl {
  public static void main(String[] args) {
    System.out.println("Main engine is online.");
    Navigation.checkStatus();
  }
}

class Navigation {
  public static void checkStatus() {
    System.out.println("Navigation system is online.");
  }
}
Output:
Main engine is online.
Navigation system is online.

Even though this is possible, the best practice is to always have **one `public` class per `.java` file**, with the filename matching the class name. It keeps your project organized and easy to navigate.
