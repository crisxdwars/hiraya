Leaving Notes in Your Code

Comments are notes that you can leave inside your code. The Java compiler completely ignores them, so they don't affect how your program runs. Their purpose is to help you and other programmers understand the code.

Good comments explain the "why" behind your code, not just the "what."

Single-Line Comments (`//`)

Single-line comments start with `//` and continue to the end of the line. They are perfect for brief explanations or labeling parts of your code.

// Calculate the total price including tax
double price = 19.99;
double taxRate = 0.12; // 12% VAT
double totalPrice = price * (1 + taxRate);

System.out.println(totalPrice);
Output:
22.3888

Multi-Line Comments (`/* ... */`)

Multi-line comments start with `/*` and end with `*/`. They can span multiple lines and are useful for longer explanations or for temporarily disabling a block of code while you're testing something else.

/*
  This block of code is temporarily disabled for testing.
  It was causing an issue with the user's final score calculation.
  We will re-enable it after debugging.

  int bonusPoints = 50;
  playerScore = playerScore + bonusPoints;
*/
System.out.println("Testing without bonus points.");
Output:
Testing without bonus points.
